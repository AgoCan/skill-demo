# Squirrel 运维平台前端布局架构

## 触发条件（When）

- 用户需要了解前端整体架构设计
- 用户需要规划或调整模块结构
- 用户需要设计布局系统
- 用户需要了解数据流管理方案
- 用户需要扩展或修改现有架构

## 执行步骤（How）

1. 确认架构需求范围（整体架构 / 模块结构 / 布局系统 / 数据流）
2. 参考相关架构设计文档
3. 提供架构建议或设计方案
4. 确保符合现有架构规范

## 输出结果（What）

- 架构设计方案或建议
- 模块结构说明
- 相关代码示例或配置参考

## 技术栈

- Vue3 + TypeScript + Vite

## 核心模块

| 模块 | 职责 |
|------|------|
| layout | 布局系统核心模块 |
| components | 通用组件库 |
| views | 页面视图（多组件模式） |
| router | 路由管理 |
| store | 状态管理（Pinia） |
| api | API 接口封装 |
| utils | 工具函数 |
| types | 类型定义 |
| lang | 国际化语言包 |
| styles | 通用样式管理 |

## 快速参考

### 数据流方向

```
用户操作 → 触发组件事件 → 调用状态管理 → 状态更新 → 组件重新渲染 → 布局变化
```

### 布局模式速查

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| 全屏模式 | 登录页、欢迎页 | 独立布局、无嵌套导航 |
| 标准模式 | 大部分管理页面 | 多层嵌套、完整导航 |
| 紧凑模式 | 数据密集型页面 | 侧边栏折叠、内容最大化 |
| 详情模式 | 详情查看页面 | 侧边栏可选、内容为主 |
