# Squirrel 配置管理页面开发规范

## 触发条件（When）

当用户需要开发配置管理相关功能时调用，包括：
- 创建配置列表页面
- 实现添加/编辑配置表单
- 开发搜索过滤功能
- 实现排序和复制功能
- 配置相关国际化

## 执行步骤（How）

### 1. 创建目录结构

```
views/Configs/
├── index.vue                 # 页面入口
├── components/
│   ├── ConfigTable.vue       # 配置列表表格
│   ├── ConfigForm.vue        # 添加/编辑表单
│   └── DeleteConfirm.vue     # 删除确认弹窗
├── composables/
│   └── useConfig.ts          # 配置相关的可组合函数
├── types/
│   └── index.ts              # 页面类型定义
└── styles/
    └── configs.scss          # 配置相关组件样式
```

### 2. 实现功能模块

按以下优先级实现：

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 模糊搜索 | ⭐⭐⭐⭐ | 按配置键和值实时过滤 |
| 快速复制 | ⭐⭐⭐⭐ | 一键复制配置值到剪贴板 |
| 排序功能 | ⭐⭐⭐⭐ | 按键/值升序降序排列 |
| 添加配置 | ⭐⭐⭐ | 创建新配置项 |
| 编辑配置 | ⭐⭐⭐ | 修改现有配置 |
| 删除配置 | ⭐⭐⭐ | 带确认的删除功能 |

### 3. 遵循设计规范

- 使用项目统一的配色方案（主色 #4fc3f7）
- 表格行布局，不是方块卡片
- 操作按钮使用图标按钮
- 遵循 Squirrel 紧凑风格页面设计规范（调用 `compact-page-style` skill）

## 输出结果（What）

完整的配置管理页面，包含：
- 配置列表展示（支持搜索、排序）
- 添加/编辑配置功能
- 快速复制功能
- 删除确认功能
- 完整的国际化支持

## API 接口速查

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/config` | GET | 获取配置列表 |
| `/api/v1/config/{id}` | GET | 获取配置详情 |
| `/api/v1/config` | POST | 创建配置 |
| `/api/v1/config/{id}` | POST | 更新配置 |
| `/api/v1/config/{id}` | DELETE | 删除配置 |

详细接口文档请参考 [api.md](./source/api.md)。

## 最佳实践

1. **列表布局**：使用表格行布局，不是方块卡片
2. **状态管理**：使用响应式数据管理弹窗显示状态
3. **错误处理**：API 调用添加 try-catch 处理
4. **国际化**：所有文本使用 $t() 函数包裹
5. **表单验证**：配置键和配置值都为必填项
6. **组件复用**：参考 `component-reusability` skill 设计可复用组件
7. **搜索优化**：使用 computed 实现实时过滤，避免不必要的重新渲染
8. **排序优化**：支持多字段排序，提供直观的排序图标
